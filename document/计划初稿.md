# WeLink 项目计划书 (初稿)

## 1. 项目愿景
构建一个具备**高并发、高可用**能力的分布式即时通讯与内容社区系统（简化版 QQ），支持海量用户在线互动、即时聊天及内容分享。

## 2. 微服务架构划分 

采用**领域驱动设计 (DDD)** 思想，将系统拆分为以下 6 个核心微服务：

| 服务名称 | 服务标识 | 核心职责 (Bounded Context) | 数据归属 |
| :--- | :--- | :--- | :--- |
| **Gateway Service** | `gateway` | 统一入口、鉴权(JWT)、限流、跨域处理。 | 无 (仅转发) |
| **User Service** | `user` | 用户注册、登录、个人信息管理、积分。 | 用户库 (`user_db`) |
| **Relation Service** | `relation` | 好友申请、好友列表、群组创建与管理。 | 关系库 (`relation_db`) |
| **Content Service** | `content` | 博客文章的 CRUD、动态(Feed)发布、评论、点赞。 | 内容库 (`content_db`) |
| **Message Service** | `message` | 消息持久化存储、历史消息查询、离线消息拉取。 | 消息库 (`message_db`) |
| **IM Connector** | `im` | **核心难点**。WebSocket 长连接管理、心跳检测、消息实时推送。 | 无 (状态存 Redis) |

## 3. 技术栈选型 

### 3.1 后端核心技术
| 类别 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **开发语言** | **Java 17** | 利用 Record、Text Blocks 等新特性。 |
| **主框架** | **Spring Boot 3.x** | 基于 Jakarta EE 9+。 |
| **微服务套件** | **Spring Cloud Alibaba 2023.x** | Nacos (注册/配置), Sentinel (流控), Seata (事务)。 |
| **ORM 框架** | **MyBatis-Plus** | 简化单表 CRUD，支持多租户和动态表名。 |
| **即时通讯** | **Netty 4.1** | 高性能 NIO 框架，支撑百万级长连接。 |
| **API 风格** | **RESTful** | 对外 RESTful，微服务间调用。 |

### 3.2 中间件生态 
| 中间件 | 选型 | 用途 |
| :--- | :--- | :--- |
| **注册/配置中心** | **Nacos** | 服务发现、动态配置管理。 |
| **分布式缓存** | **Redis Cluster** | 缓存、分布式锁、Session、IM 路由表。 |
| **消息队列** | **Kafka** | 削峰填谷、异步解耦 (消息发送、动态写扩散)。 |
| **分布式搜索** | **Elasticsearch** | 博客文章搜索、用户搜索。 |
| **对象存储** | **MinIO** | 存储图片、附件、博客备份。(封装非阻塞IO) |
| **数据库** | **MySQL 8.0** | 业务数据存储，配合 ShardingSphere 分库分表。 |

### 3.3 运维与监控
| 工具 | 用途 |
| :--- | :--- |
| **Docker** | 容器化封装。 |
| **Prometheus** | 指标采集 (JVM, CPU, 内存)。 |
| **Grafana** | 监控大盘可视化。 |
| **SkyWalking** | 分布式链路追踪 (排查慢接口)。 |

## 4. 核心功能实现清单(初稿)

### 4.1 用户与社交 
*   [ ] **注册/登录**：手机号/邮箱注册，JWT 无状态认证。
*   [ ] **好友系统**：搜索用户、发送/接受好友申请、好友列表、黑名单。
*   [ ] **群组系统**：创建群、邀请入群、踢人、群公告。

### 4.2 即时通讯  
*   [ ] **WebSocket 接入**：基于 Netty 实现 WebSocket 服务器。
*   [ ] **私聊消息**：点对点实时通讯。
*   [ ] **群聊消息**：点对多广播。
*   [ ] **消息可靠性**：
    *   已读回执 (Read Receipt)。
    *   离线消息存储 (用户上线后拉取)。
*   [ ] **在线状态**：实时显示好友在线/离线。

### 4.3 内容社区  
*   [ ] **博客系统**：Markdown 编辑器、发布、分类、标签。
*   [ ] **社交动态 (Feed)**：
    *   **发布**：发布文字、图片动态。
    *   **Timeline**：好友动态流 (推拉结合模式)。
    *   **互动**：点赞、评论 (评论回复)。
*   [ ] **搜索**：基于 ES 的博客全文检索。

### 4.4 系统支撑 
*   [ ] **文件上传**：图片/附件上传至 MinIO，返回 URL。
*   [ ] **接口限流**：基于 Sentinel 的网关级限流和接口级熔断。
*   [ ] **幂等性**：防止消息重发、防止重复提交。

## 5. 开发里程碑 

### Phase 1: 基础设施与骨架
*   搭建 Spring Cloud Alibaba 父工程。
*   部署 Docker 环境 (MySQL, Redis, Nacos, Kafka)。
*   实现 Gateway 网关和 User 服务 (注册/登录)。

### Phase 2: 社交与内容 
*   实现 Relation 服务 (好友/群组)。
*   实现 Content 服务 (博客 CRUD)。
*   实现动态发布与 Redis Timeline 构建。

### Phase 3: IM 核心攻坚 
*   **Netty 入门**：搭建 WebSocket Server。
*   **连接管理**：实现用户登录鉴权、Channel 管理。
*   **消息路由**：实现 Redis 路由表，完成跨服务器私聊。
*   **群聊广播**：实现群消息分发。

### Phase 4: 高并发与高可用 
*   集成 Sentinel 实现限流熔断。
*   集成 Seata 解决分布式事务 (如：发动态加积分)。
*   压力测试 (JMeter) 与性能调优。
*   部署 SkyWalking 进行链路追踪。

## 6. 难点与解决方案 

| 难点 | 解决方案 |
| :--- | :--- |
| **IM 分布式会话** | Redis 存储 `UserId -> ServerAddress` 映射，实现路由转发。 |
| **消息不丢失** | Kafka 持久化 + MySQL 落库 + ACK 确认机制。 |
| **动态流性能** | **写扩散 (Push)**：发布动态时写入所有好友的 Timeline (Redis ZSet)。 |
| **博客热点数据** | Redis + Caffeine 二级缓存，解决单 Key 过热问题。 |
| **服务雪崩** | Sentinel 熔断降级，隔离舱壁模式。 |
