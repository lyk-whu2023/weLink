# WeLink 项目启动指南

## 1. 项目简介

WeLink 是一个基于 Java 17 + Spring Boot 3.x 的即时通讯系统，支持私聊、群聊、好友管理、群组管理等核心功能。系统采用 Netty 实现 WebSocket 实时通信，通过 Redis 和 Kafka 支持分布式部署。

**技术栈：**
- **后端：** Java 17、Spring Boot 3.2.5、Netty 4.1、MyBatis-Plus 3.5.5
- **中间件：** MySQL 8.0、Redis 7.x、Kafka 3.x、MinIO
- **前端：** Vue 3、Element Plus、Pinia、WebSocket

## 2. 前置环境要求

在启动项目之前，请确保以下环境已正确安装并运行：

| 组件 | 版本要求 | 用途 | 默认端口 |
|------|----------|------|----------|
| JDK | 17+ | Java 运行环境 | - |

## 3. 中间件配置

### 3.1 MySQL 数据库初始化

```bash
# 登录 MySQL
mysql -u root -p

# 执行初始化脚本
mysql -u root -p < src/main/resources/sql/init.sql
Get-Content -Raw .\main\resources\sql\init.sql | mysql -u root -p
```

初始化脚本会创建以下内容：
- 数据库 `welink`
- 用户表 `user`
- 好友关系表 `friend_relation`
- 群组表 `group_info`
- 群成员表 `group_member`
- 消息表 `message`

### 3.2 Redis 配置

Redis 使用默认配置即可，无需特殊设置。确保 Redis 服务正在运行：

```bash
redis-cli ping
# 输出: PONG
```

### 3.3 Kafka 配置

#### 3.3.1 启动 Kafka

```bash
# 启动 Zookeeper（Kafka 依赖）
zookeeper-server-start.sh config/zookeeper.properties

# 启动 Kafka
kafka-server-start.sh config/server.properties
```

#### 3.3.2 创建主题

```bash
# 创建私聊消息主题
kafka-topics.sh --create \
  --bootstrap-server localhost:9092 \
  --topic im-private-message \
  --partitions 3 \
  --replication-factor 1

# 创建群聊消息主题
kafka-topics.sh --create \
  --bootstrap-server localhost:9092 \
  --topic im-group-message \
  --partitions 3 \
  --replication-factor 1
```

### 3.4 MinIO 配置

#### 3.4.1 启动 MinIO

```bash
# 启动 MinIO 服务
minio server /data --console-address ":9001"
```

#### 3.4.2 创建存储桶

访问 MinIO 控制台：http://localhost:9001

- 默认用户名：`minioadmin`
- 默认密码：`minioadmin`

在控制台中创建名为 `welink` 的存储桶，访问权限设置为 `public`。

## 4. 后端启动

### 4.1 配置文件检查

打开 `src/main/resources/application.properties`，根据实际情况修改以下配置：

```properties
# MySQL 连接配置
spring.datasource.url=jdbc:mysql://localhost:3306/welink?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
spring.datasource.username=root
spring.datasource.password=root

# Redis 连接配置
spring.data.redis.host=localhost
spring.data.redis.port=6379
spring.data.redis.password=

# Kafka 连接配置
spring.kafka.bootstrap-servers=localhost:9092

# MinIO 连接配置
welink.minio.endpoint=http://localhost:9000
welink.minio.access-key=minioadmin
welink.minio.secret-key=minioadmin

# JWT 密钥（生产环境请修改为更复杂的密钥）
welink.jwt.secret=WeLinkJwtSecretKey2026VeryLongSecretKeyForSecurity

# 服务端口
server.port=8080
welink.websocket.port=8081
```

### 4.2 编译项目

```bash
# 进入项目根目录
cd WeLink

# 清理并编译
mvn clean package -DskipTests
```

### 4.3 启动服务

```bash
# 方式一：使用 Maven 启动（开发环境）
mvn spring-boot:run

# 方式二：使用 JAR 包启动（生产环境）
java -jar target/WeLink-0.0.1-SNAPSHOT.jar
```

### 4.4 验证后端启动

```bash
# 检查 REST API 健康状态
curl http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"test123"}'
# 应返回 {"code":200,"message":"success","data":null}

# 检查 WebSocket 端口
telnet localhost 8081
# 应能成功连接
```

**启动成功标志：**
- 控制台输出 `Started WeLinkApplication in X.XXX seconds`
- REST API 端口 8080 可访问
- WebSocket 端口 8081 可连接

## 5. 前端启动

### 5.1 安装依赖

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install
```

### 5.2 配置代理

前端开发服务器已配置代理，将 `/api` 请求转发到后端 `http://localhost:8080`。

配置文件：`vite.config.js`

```javascript
server: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

### 5.3 启动开发服务器

```bash
npm run dev
```

启动成功后访问：http://localhost:3000

### 5.4 构建生产版本

```bash
npm run build
```

构建产物输出到 `dist/` 目录，可部署到 Nginx 或其他静态服务器。

## 6. 功能验证

### 6.1 用户注册

1. 访问 http://localhost:3000
2. 切换到"注册"标签
3. 输入用户名（4-20字符）和密码（6-32字符）
4. 点击"注册"按钮

### 6.2 用户登录

1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后自动跳转到聊天页面

### 6.3 添加好友

1. 点击左侧导航栏"联系人"
2. 点击"添加好友"按钮
3. 输入目标用户 ID
4. 点击"发送申请"

### 6.4 创建群组

1. 点击左侧导航栏"联系人"
2. 切换到"群组"标签
3. 点击"创建群组"按钮
4. 输入群组名称和公告
5. 点击"创建"

### 6.5 发送消息

1. 点击左侧导航栏"消息"
2. 在联系人页面点击好友或群组
3. 自动跳转到聊天页面
4. 在输入框输入消息内容
5. 按 `Ctrl+Enter` 或点击"发送"按钮

## 7. 常见问题

### 7.1 后端启动失败

**问题：** `Connection refused: connect`

**原因：** MySQL、Redis 或 Kafka 未启动

**解决方案：**
```bash
# 检查 MySQL
systemctl status mysql

# 检查 Redis
systemctl status redis

# 检查 Kafka
systemctl status kafka
```

### 7.2 前端无法连接后端

**问题：** 登录失败，提示网络错误

**原因：** 后端未启动或端口不匹配

**解决方案：**
1. 确认后端已在 8080 端口启动
2. 检查 `vite.config.js` 中的代理配置
3. 检查浏览器控制台网络请求

### 7.3 WebSocket 连接失败

**问题：** 登录后无法建立 WebSocket 连接

**原因：** WebSocket 端口未开放或防火墙拦截

**解决方案：**
```bash
# 检查 8081 端口是否监听
netstat -an | grep 8081

# 检查防火墙规则
sudo ufw status
```

### 7.4 Kafka 主题不存在

**问题：** 消息发送失败，提示主题不存在

**原因：** Kafka 主题未创建

**解决方案：** 参考 [3.3.2 创建主题](#332-创建主题) 章节创建所需主题。

### 7.5 MinIO 上传失败

**问题：** 文件上传失败

**原因：** MinIO 存储桶未创建或权限不足

**解决方案：**
1. 登录 MinIO 控制台 http://localhost:9001
2. 确认 `welink` 存储桶已创建
3. 检查存储桶访问权限设置为 `public`

## 8. 项目结构

```
WeLink/
├── src/
│   ├── main/
│   │   ├── java/com/epsilon/welink/
│   │   │   ├── common/          # 公共模块（配置、工具类、异常处理）
│   │   │   ├── gateway/         # 网关模块（JWT 拦截器）
│   │   │   ├── user/            # 用户模块（注册、登录）
│   │   │   ├── relation/        # 关系模块（好友、群组）
│   │   │   ├── message/         # 消息模块（历史消息、离线消息）
│   │   │   └── im/              # IM 模块（WebSocket、消息推送）
│   │   └── resources/
│   │       ├── application.properties  # 应用配置
│   │       └── sql/init.sql           # 数据库初始化脚本
│   └── test/                    # 测试代码
├── frontend/                    # 前端项目
│   ├── src/
│   │   ├── api/                 # API 接口封装
│   │   ├── router/              # 路由配置
│   │   ├── stores/              # Pinia 状态管理
│   │   ├── utils/               # 工具类（WebSocket、请求）
│   │   └── views/               # 页面组件
│   ├── package.json
│   └── vite.config.js
├── pom.xml                      # Maven 依赖配置
└── document/                    # 项目文档
```
