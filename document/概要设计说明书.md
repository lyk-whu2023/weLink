## 2 总体设计

### 2.1 需求规定

本系统的主要输入输出项目、处理的功能性能要求详见《软件需求说明书》。以下为主要需求摘要：

**输入项目：**

- 用户注册/登录信息
- 好友申请/处理请求
- 群组创建/管理请求
- 私聊/群聊消息
- 文件上传请求

**输出项目：**

- 认证 Token
- 好友/群组列表
- 实时消息推送
- 历史消息查询结果
- 文件访问 URL

**功能性能要求：**

- 并发用户数：10,000+（单实例）
- 消息延迟：< 500ms（95%）
- 消息可靠性：99.99%
- 系统可用性：99.9%

### 2.3 基本设计概念和处理流程

**设计概念：**

系统采用单体架构，将核心功能模块整合在单一应用中，通过合理的模块划分和接口设计保持清晰的边界。系统支持多实例分布式部署，通过 Redis 和 Kafka 解决分布式环境下的会话路由和消息转发问题。

**核心处理流程：**

**1. 用户登录流程：**

```
客户端发起登录请求
        │
        ▼
  验证用户名密码
        │
        ▼
  生成 JWT Token
        │
        ▼
  返回 Token 给客户端
        │
        ▼
  客户端建立 WebSocket 连接
        │
        ▼
  验证 Token，建立会话
        │
        ▼
  记录路由表到 Redis
        │
        ▼
  推送离线消息
```

**2. 私聊消息流程：**

```
用户 A 发送消息
        │
        ▼
  生成 msgId，持久化到 MySQL
        │
        ▼
  推送消息到 Kafka
        │
        ▼
  查询用户 B 在线状态
        │
   ┌────┴────┐
   │         │
 B 在线    B 离线
   │         │
   ▼         ▼
查询路由表  标记离线消息
   │
┌──┴──┐
│     │
同实例 不同实例
│     │
▼     ▼
直接推送 Kafka 转发
        │
        ▼
    目标实例推送
```

**3. 群聊消息流程：**

```
用户 A 发送群消息
        │
        ▼
  生成 msgId，持久化到 MySQL
        │
        ▼
  推送消息到 Kafka
        │
        ▼
  查询群成员列表
        │
        ▼
  过滤在线成员
        │
   ┌────┼────┐
   │    │    │
同实例 不同实例 离线成员
   │    │    │
   ▼    ▼    ▼
直接推送 Kafka转发 标记离线
```

## 4 运行设计

### 4.1 运行模块组合

**场景一：用户登录**

|  步骤 |       模块      |           操作           |
| :-: | :-----------: | :--------------------: |
|  1  |    Gateway    | 接收登录请求，路由到 User Module |
|  2  |  User Module  |         验证用户名密码        |
|  3  |  User Module  |      生成 JWT Token      |
|  4  | Common Module |      缓存用户信息到 Redis     |
|  5  |    Gateway    |      返回 Token 给客户端     |

**场景二：发送私聊消息**

|  步骤 |       模块       |             操作            |
| :-: | :------------: | :-----------------------: |
|  1  |    IM Module   |      接收 WebSocket 消息      |
|  2  |  Common Module |          验证 Token         |
|  3  | Message Module |        持久化消息到 MySQL       |
|  4  |  Common Module |        推送消息到 Kafka        |
|  5  |    IM Module   |          查询目标用户路由         |
|  6  |    IM Module   | 推送消息（同实例）或 Kafka 转发（不同实例） |

**场景三：发送群聊消息**

|  步骤 |        模块       |        操作       |
| :-: | :-------------: | :-------------: |
|  1  |    IM Module    | 接收 WebSocket 消息 |
|  2  |  Common Module  | 验证 Token 和群成员权限 |
|  3  |  Message Module |   持久化消息到 MySQL  |
|  4  |  Common Module  |   推送消息到 Kafka   |
|  5  | Relation Module |     查询群成员列表     |
|  6  |    IM Module    |      批量推送消息     |

### 4.2 运行控制

**启动控制：**

|  步骤 |             操作             |
| :-: | :------------------------: |
|  1  | 启动 MySQL、Redis、Kafka、MinIO |
|  2  |        启动 WeLink 应用        |
|  3  |         应用初始化数据库连接池        |
|  4  |       应用初始化 Redis 连接       |
|  5  |     应用初始化 Kafka 生产者/消费者    |
|  6  |   启动 Netty WebSocket 服务器   |
|  7  |        注册服务实例到 Redis       |

**停止控制：**

|  步骤 |          操作         |
| :-: | :-----------------: |
|  1  |        接收停止信号       |
|  2  |       停止接收新连接       |
|  3  | 等待现有连接处理完成（最多 30 秒） |
|  4  |  关闭所有 WebSocket 连接  |
|  5  |     清理 Redis 路由表    |
|  6  |   关闭 Kafka 生产者/消费者  |
|  7  |       关闭数据库连接池      |
|  8  |         应用退出        |

### 4.3 运行时间

| 运行模块组合 |  资源占用时间 |        说明        |
| :----: | :-----: | :--------------: |
|  用户登录  | < 200ms | 包含密码验证和 Token 生成 |
| 发送私聊消息 |  < 50ms |     不包含网络传输时间    |
| 发送群聊消息 | < 100ms |    不包含群成员推送时间    |
| 查询历史消息 | < 100ms |     查询 50 条消息    |
|  文件上传  |   < 2s  |     上传 1MB 文件    |

### 6.1 出错信息

|      错误类型      |  错误码 |      错误信息      |      处理方法     |
| :------------: | :--: | :------------: | :-----------: |
|     参数校验失败     |  400 |  请求参数错误：{具体字段} |    检查请求参数格式   |
|      认证失败      |  401 | 未认证或 Token 已过期 |  重新登录获取 Token |
|      权限不足      |  403 |    无权限执行此操作    |     联系管理员     |
|      资源不存在     |  404 |    请求的资源不存在    |    检查资源 ID    |
|      资源冲突      |  409 |   用户名已存在/已是好友  |  更换用户名/检查好友状态 |
|     请求过于频繁     |  429 |  请求过于频繁，请稍后重试  |     降低请求频率    |
|     服务器内部错误    |  500 |     服务器内部错误    |   联系管理员，查看日志  |
|     数据库连接失败    | 5001 |     数据库连接失败    |   检查数据库服务状态   |
|   Redis 连接失败   | 5002 |    缓存服务连接失败    | 检查 Redis 服务状态 |
|   Kafka 连接失败   | 5003 |    消息队列连接失败    | 检查 Kafka 服务状态 |
| WebSocket 连接断开 | 1000 |      连接已断开     |    客户端自动重连    |
|      心跳超时      | 1001 |   心跳超时，连接已断开   |    客户端自动重连    |

### 6.2 补救措施

**a. 后备技术：**

|    场景    |   后备技术  |         说明         |
| :------: | :-----: | :----------------: |
|   数据库故障  |  定期备份恢复 |    每日全量备份，实时增量备份   |
| Redis 故障 | 降级到本地缓存 |   部分功能降级，不影响核心通讯   |
| Kafka 故障 |  本地消息队列 | 消息暂存本地，Kafka 恢复后补发 |

**b. 降效技术：**

|     场景    |    降效措施    |     说明     |
| :-------: | :--------: | :--------: |
|   单实例故障   | 负载均衡剔除故障实例 |  其他实例继续服务  |
| Redis 不可用 |  在线状态查询降级  |  显示默认离线状态  |
| Kafka 不可用 |   跨实例消息延迟  | 消息暂存，恢复后转发 |

**c. 恢复及再启动技术：**

|      场景      |  恢复技术 |     说明     |
| :----------: | :---: | :--------: |
|     服务崩溃     |  自动重启 | 系统监控服务自动拉起 |
|    数据库连接断开   |  自动重连 |  连接池自动重连机制 |
| WebSocket 断开 | 客户端重连 |  指数退避重连策略  |
