本文档旨在详细描述 WeLink 即时通讯系统的软件需求，为系统设计、开发、测试和验收提供依据。本文档的预期读者包括：

### 1.2 范围

a. 待开发的软件系统名称：WeLink 即时通讯系统

**软件将实现：**

- 用户注册、登录和个人信息管理

## 2 项目概述

### 2.1 产品描述

WeLink 即时通讯系统是一个基于 Web 的实时通讯平台，旨在为用户提供稳定、高效的私聊和群聊服务。系统采用 Java 17 + Spring Boot 3.x 作为后端技术栈，Netty 作为 WebSocket 服务器，Redis 和 Kafka 作为中间件支撑分布式部署。

**开发意图：**

- 提供一个简化版的 QQ 式即时通讯体验
- 支持大规模用户同时在线
- 保证消息的可靠性和实时性

**应用目标：**

- 个人用户之间的即时通讯
- 群组讨论和协作
- 文件共享

**作用范围：**

- Web 浏览器访问
- 后续支持浏览器嵌套客户端

**系统组成：**

```
┌─────────────────────────────────────────────────┐
│                    客户端 (Web)                   │
└─────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│                  Nginx 负载均衡                   │
└─────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│              WeLink 后端单体应用                  │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │
│  │Gateway│ │ User │ │Relation│ │Message│ │  IM  │ │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ │
└─────────────────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐

    ┌───────┐       ┌───────┐       ┌───────┐
    │ MySQL │       │ Redis │       │ Kafka │
    └───────┘       └───────┘       └───────┘
```

|             假设            |         依据        |            影响           |
| :-----------------------: | :---------------: | :---------------------: |

## 3 具体需求

### 3.1 功能需求

#### 3.1.1 用户注册与登录

**a. 引言**

提供用户注册和登录功能，是系统的基础功能。用户通过注册创建账号，通过登录获取访问权限。

**b. 输入**

|   输入项   |  输入源 |       有效范围      |  备注  |
| :-----: | :--: | :-------------: | :--: |
|   用户名   | 用户输入 | 4-20 字符，字母数字下划线 |  唯一  |
|    密码   | 用户输入 |     6-32 字符     | 加密传输 |
|  邮箱（可选） | 用户输入 |      有效邮箱格式     |  唯一  |
| 手机号（可选） | 用户输入 |     有效手机号格式     |  唯一  |

**c. 加工**

1. **注册流程：**
   - 校验输入参数格式
   - 检查用户名唯一性
   - 密码使用 BCrypt 加密
   - 创建用户记录
   - 返回注册成功
2. **登录流程：**
   - 校验输入参数格式
   - 查询用户记录
   - 验证密码
   - 生成 JWT Token（有效期 2 小时）
   - 生成 Refresh Token（有效期 7 天）
   - 缓存用户信息到 Redis
   - 返回 Token
3. **Token 刷新：**
   - 验证 Refresh Token 有效性
   - 生成新的 JWT Token
   - 返回新 Token

**d. 输出**

|  输出项 | 输出目的地 |  格式  |      备注     |
| :--: | :---: | :--: | :---------: |
| 注册结果 |  客户端  | JSON |   成功/失败及原因  |
| 登录结果 |  客户端  | JSON | Token 及用户信息 |
| 错误信息 |  客户端  | JSON |    错误码和描述   |

#### 3.1.3 群组管理系统

**a. 引言**

提供群组创建、管理和成员管理功能，支持多人协作和讨论。

**b. 输入**

|    输入项    |  输入源 |      有效范围      |  备注  |
| :-------: | :--: | :------------: | :--: |
|    群组名称   | 用户输入 |     2-50 字符    |   -  |
| 群成员 ID 列表 | 用户输入 |     有效用户 ID    | 初始成员 |
|    群公告    | 用户输入 |    0-500 字符    |  可选  |
|    操作类型   | 用户选择 | 创建/邀请/踢出/退出/修改 |   -  |

**c. 加工**

1. **创建群组：**
   - 校验群组名称
   - 创建群组记录
   - 添加创建者为群主
   - 批量添加初始成员
   - 推送群邀请通知
2. **邀请成员：**
   - 验证操作权限（群主/管理员）
   - 检查成员是否已在群中
   - 添加群成员记录
   - 推送群邀请通知
3. **踢出成员：**
   - 验证操作权限
   - 不能踢出群主
   - 删除群成员记录
   - 推送踢出通知
4. **退出群组：**
   - 普通成员直接退出
   - 群主退出需转移群主权限
   - 删除群成员记录

**d. 输出**

|  输出项  | 输出目的地 |     格式    |      备注     |
| :---: | :---: | :-------: | :---------: |
|  群组列表 |  客户端  |  JSON 数组  |   用户加入的群组   |
| 群成员列表 |  客户端  |  JSON 数组  |    群内成员信息   |
|  群信息  |  客户端  |    JSON   |    群组详细信息   |
|  操作结果 |  客户端  |    JSON   |    成功/失败    |
|  实时通知 |  客户端  | WebSocket | 群邀请/踢出/公告更新 |

#### 3.1.4 私聊消息系统

**a. 引言**

提供用户之间点对点的实时消息通讯功能，是系统的核心功能。

**b. 输入**

|   输入项   |  输入源 |      有效范围      |  备注  |
| :-----: | :--: | :------------: | :--: |
|   消息内容  | 用户输入 |    0-2000 字符   | 文本消息 |
| 目标用户 ID | 系统获取 |     有效用户 ID    |   -  |
|   消息类型  | 系统设置 | 1-文本/2-图片/3-文件 |   -  |
|  消息 ID  | 系统生成 |      UUID      | 唯一标识 |

**c. 加工**

1. **消息发送：**
   - 生成消息唯一 ID
   - 校验消息内容
   - 持久化消息到 MySQL
   - 推送消息到 Kafka
   - 查询目标用户在线状态
   - 根据路由表推送消息
2. **消息路由：**
   - 同实例：直接通过 WebSocket 推送
   - 不同实例：通过 Kafka 转发到目标实例
   - 目标实例推送给用户
3. **已读回执：**
   - 接收方已读后发送确认
   - 更新消息状态为已读
   - 通知发送方
4. **离线消息：**
   - 目标用户离线时标记为离线消息
   - 用户上线时拉取离线消息

**d. 输出**

|   输出项  | 输出目的地 |       格式       |    备注   |
| :----: | :---: | :------------: | :-----: |
|  实时消息  |  客户端  | WebSocket JSON |   消息内容  |
|  发送确认  |  客户端  | WebSocket JSON |  消息已送达  |
|  已读回执  |  客户端  | WebSocket JSON |   消息已读  |
| 离线消息列表 |  客户端  |    REST JSON   | 用户上线时拉取 |

#### 3.1.5 群聊消息系统

**a. 引言**

提供群组内的多人实时消息通讯功能，支持群广播。

**b. 输入**

|  输入项  |  输入源 |      有效范围      |  备注  |
| :---: | :--: | :------------: | :--: |
|  消息内容 | 用户输入 |    0-2000 字符   | 文本消息 |
| 群组 ID | 系统获取 |     有效群组 ID    |   -  |
|  消息类型 | 系统设置 | 1-文本/2-图片/3-文件 |   -  |
| 消息 ID | 系统生成 |      UUID      | 唯一标识 |

**c. 加工**

1. **消息发送：**
   - 生成消息唯一 ID
   - 校验消息内容和发送权限
   - 持久化消息到 MySQL
   - 推送消息到 Kafka
   - 查询群成员列表
2. **消息广播：**
   - 过滤在线成员
   - 同实例成员：直接推送
   - 不同实例成员：通过 Kafka 转发
   - 离线成员：标记为离线消息
3. **大群优化：**
   - 成员数 > 100 的群采用写扩散模式
   - 提前计算推送列表
   - 异步推送，避免阻塞

**d. 输出**

|   输出项  |  输出目的地 |       格式       |    备注   |
| :----: | :----: | :------------: | :-----: |
|  实时消息  | 群成员客户端 | WebSocket JSON |   消息内容  |
|  发送确认  | 发送方客户端 | WebSocket JSON |  消息已送达  |
| 离线消息列表 |   客户端  |    REST JSON   | 用户上线时拉取 |

**a. 引言**

提供历史消息查询功能，用户可以查看之前的聊天记录。

**b. 输入**

**c. 加工**

#### 3.1.7 在线状态管理

**a. 引言**

提供用户在线状态的实时显示和更新功能。

**b. 输入**

|      输入项     |   输入源  |  有效范围  |  备注  |
| :----------: | :----: | :----: | :--: |
| WebSocket 连接 |   客户端  |  有效连接  | 上线标志 |
|      心跳包     |   客户端  | 每 30 秒 |  续期  |
|     连接断开     | 客户端/系统 |  连接关闭  | 下线标志 |

**c. 加工**

1. **上线处理：**
   - WebSocket 连接成功
   - Redis 记录在线状态
   - 设置 TTL 120 秒
   - 发布上线通知
2. **心跳续期：**
   - 每 30 秒接收心跳
   - 更新 Redis TTL
   - 60 秒未收到心跳标记为离线
3. **下线处理：**
   - 连接断开或超时
   - 清理 Redis 在线状态
   - 发布下线通知
4. **状态推送：**
   - 通过 Redis Pub/Sub 通知好友
   - 好友客户端更新显示

**d. 输出**

|   输出项  | 输出目的地 |       格式       |    备注    |
| :----: | :---: | :------------: | :------: |
|  在线状态  | 好友客户端 | WebSocket JSON |  上线/下线通知 |
| 状态查询结果 |  客户端  |    REST JSON   | 好友在线状态列表 |
